# Contribution Guidelines

Please note that this project is released with a [Contributor Code of Conduct](code-of-conduct.md). By participating in this project you agree to abide by its terms.

---

## The Pull Request Process

Ensure your pull request adheres to the following guidelines:

- Search previous suggestions before making a new one to avoid duplicates.
- Make sure the item you are adding is genuinely useful, actively maintained, and has proper documentation.
- Make an individual pull request for each suggestion.
- Use the following format: `- [Name](link) - Description.`
- The description should start with an uppercase character and end with a period.
- New categories or improvements to the existing categories are welcome.
- Check your spelling and grammar.
- Make sure your text editor is set to remove trailing whitespace.
- The pull request and commit should have a useful title.
- Link additions should be added to the bottom of the relevant category.

## Quality Standards

To be included in this list, a resource must meet the following criteria:

- **Actively maintained** — the project must have had a commit within the last 12 months.
- **Not archived** — archived GitHub repositories will not be accepted.
- **Not deprecated** — tools with official deprecation notices will not be accepted.
- **Documented** — the project must have a proper readme or documentation website.
- **Open source or free tier** — paid-only tools without a meaningful free tier will not be accepted.

## Updating Your Pull Request

Sometimes a maintainer will ask you to edit your pull request before it is included. This is normally due to spelling errors or because your PR did not match these guidelines. [Here](https://github.com/RichardLitt/knowledge/blob/master/github/amending-a-commit-guide.md) is a write-up on how to change a pull request and the different ways you can do that.
